2.python学习_进程与线程（2022-9-9）

一个简单的例子

进程就像我们的笔记本打开一个浏览器，打开一个word，这是两个不同的进程。
线程则是我们在word中，可以打字，同时给我们拼写检查等，这就是子进程，也是线程。
并且一个进程至少会有一个线程。

那么回到Python编程，我们想要同时执行多个任务，可以有两个解决方案：

  1. 多个进程，每个进程有一个线程。
  2. 一个进程，这个进程包含多个线程。

多进程

首先注意一个问题，在不同的操作系统，python启动多线程的方式也是有区别的。
linux系统
在linux系统上，存在一个fork（）函数的调用，区别于普通函数，fork函数调用一次，返回两次，linux会自动将当前进程复制一份（子进程），分别在父子进程中返回。
子进程返回为0，父进程返回子进程的ID，那么这样一个父进程就会fork出很多的子进程，父进程只要统计子进程的ID，子进程只要调用getpid就可以找到父进程的ID。
Python的例子
import os
print('Process (%s) start...) % os.getpid
pid = os.fork()
if pid == 0:
    print('I am child process (%s) and my parent is %s.' % (os.getpid(), os.getppid()))
else:
    print('I (%s) just created a child process (%s).' % (os.getpid(), pid))
运行结果
Process (876) start…
I (876) just created a child process (877).
I am child process (877) and my parent is 876.

Windows系统
win系统于linux不同，在win系统执行多进程需要使用multiprocessing模块。

from multiprocessing import Process
import os
def run_proc(name):
print('Run child process %s (%s)...' % (name, os.getpid()))

if name=='main':
    print('Parent process %s.' % os.getpid())；#打印父进程的ID
    p = Process(target=run_proc, args=('test',))；#将唤醒子进程的函数放到p中
    print('Child process will start.')
    p.start()；#打印子进程的ID
    p.join()
    print('Child process end.')

上述代码运行的结果为：
Parent process 928.
Child process will start.
Run child process test (929)…
Process end.
这其实比较好理解，我自己的理解放到了上述代码的注释中。唯一需要说明的是join函数，它会等待子进程运行结束后，在继续向下运行。

进程池pool
应用于需要启动大量的子进程的需求（批量）。

from multiprocessing import Pool
import os, time, random

def long_time_task(name):
    print('Run task %s (%s)...' % (name, os.getpid()))
    start = time.time()
    time.sleep(random.random() * 3)
    end = time.time()
    print('Task %s runs %0.2f seconds.' % (name, (end - start)))

if name=='main':
    print('Parent process %s.' % os.getpid())
    p = Pool(4)
    for i in range(5):
        p.apply_async(long_time_task, args=(i,))
    print('Waiting for all subprocesses done...')
    p.close()
    p.join()
    print('All subprocesses done.')

上述函数结果为
Parent process 669.
Waiting for all subprocesses done…
Run task 0 (671)…
Run task 1 (672)…
Run task 2 (673)…
Run task 3 (674)…
Task 2 runs 0.14 seconds.
Run task 4 (673)…
Task 1 runs 0.27 seconds.
Task 3 runs 0.86 seconds.
Task 0 runs 1.41 seconds.
Task 4 runs 1.91 seconds.
All subprocesses done.
不难理解的是，Pool会批量产生子进程，但是pool的参数会限制你同时运行子线程的个数。这也就是为什么0123先运行后，4才会再运行。

外部进程作为子进程
很多时候，创建子进程是为了更方便运行其他进程（比如一些shell命令），我们通过subprocess模块启动子进程，并控制输入输出。当然像os.system这类的函数也是可以的
